# CLAUDE.md - Gym CLI Project

## Проект
**Gym CLI** - дневник тренировок в командной строке. Решает проблему потерянных блокнотов и неразборчивых записей.

## Стек
- Python
- Click (CLI framework)
- SQLite (хранение данных)

## Команды MVP
```bash
gym add "жим лёжа" 80kg 8reps 3sets      # добавить упражнение
gym add "присед" 100kg 5x4 --note "..."  # с заметкой
gym today                                 # что делал сегодня
gym max "жим лёжа"                       # личный максимум
gym progress "присед"                    # прогресс за 3 месяца
gym last "становая"                      # когда последний раз
```

## Критерии оценки экзамена

| Критерий | Баллы | Требование |
|----------|-------|------------|
| Работоспособность MVP | 2-5 | Запускается и решает заявленную проблему |
| Эволюция (коммиты) | 2-5 | 10+ осмысленных коммитов, видно развитие |
| Тесты | 2-5 | pytest-bdd с .feature файлами, тесты ДО кода |
| Документация | 2-5 | README + CLAUDE.md + инструкция по запуску |
| Vibe Coding подход | 2-5 | BDD как спецификация, AI как соавтор |

## Важные напоминания

### TDD/BDD подход
- **Сначала тесты, потом код!**
- Писать .feature файлы перед реализацией
- pytest-bdd для BDD тестирования

### Коммиты
- Минимум 10+ осмысленных коммитов
- Каждый коммит = логический шаг развития
- Понятные commit messages

## Workflow
1. Написать feature файл (спецификация)
2. Запустить тест (должен упасть)
3. Написать код
4. Тест проходит
5. Коммит
6. Повторить
